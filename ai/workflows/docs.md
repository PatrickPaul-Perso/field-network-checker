# Documentation And UI Text Workflow

Use this workflow when the task is primarily about wording, README content, labels, descriptions, or user-facing guidance.

## Goal

Produce concise, accurate, easy-to-scan content aligned with the product’s field-ready tone.

## Principles

- Prefer short sentences.
- Prefer direct wording.
- Keep labels easy to scan.
- Avoid jargon when simpler terms work.
- Match the real product behavior.
- Keep technical notes accurate and compact.

## For README Updates

Cover:
- what the project does
- how to run it
- main behaviors
- configuration basics
- any validation steps
- any known constraints

## For UI Text

Check:
- state labels
- button labels
- helper text
- form field labels
- status messages
- error messages

## For Release Notes Or Change Summaries

State:
- what changed
- why it matters
- any user action needed
- any behavior to watch

## Review Checklist

Before finalizing:
- is the wording correct
- is it shorter than before where possible
- does it match actual behavior
- does it remove ambiguity
- is it easy for an operator to scan quickly

## Output Template

### Audience
Who this text is for.

### Intent
What the text needs to help them do.

### Draft
The proposed text.

### Notes
Any assumptions or limitations.
